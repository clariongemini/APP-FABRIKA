# Çoklu Dil (i18n) Standardı

Katman: **3, 6** | Sorumlu: Android Elite Ajanı

## Kural

**Hard-coded string kesinlikle yasaktır.** Tüm metinler locale JSON dosyalarından gelir.

## Dosya Yapısı

```text
app/src/main/assets/locales/
├── tr.json
├── en.json
└── (gelecek: de.json, ar.json ...)
```

## JSON Formatı

```json
{
  "app_name": "Uygulama Adı",
  "home_title": "Ana Sayfa",
  "error_network": "Bağlantı hatası. Tekrar deneyin.",
  "premium_trial_cta": "7 Gün Ücretsiz Dene"
}
```

Anahtarlar: `snake_case`, namespace prefix önerilir (`home_`, `settings_`, `error_`).

## Kotlin Kullanımı

```kotlin
// core/i18n/StringResources.kt
@Composable
fun localized(key: String): String {
    val locale = LocalLocale.current
    return remember(key, locale) { LocaleManager.getString(key, locale) }
}

// Kullanım
Text(text = localized("home_title"))
```

## Denetim

`scripts/validate-code.sh` Kotlin dosyalarında Türkçe karakterli `Text("...")` arar — ihlal = build fail.

## Eksik Anahtar Politikası

- Geliştirme: `en` fallback
- Release: tüm anahtarlar `tr` + `en` dolu olmalı (Auditor reddeder)

## Android strings.xml

Sadece sistem gerektiren sabitler (`app_name` manifest için) `strings.xml`'de kalabilir. UI metinleri JSON'da.
