# Design Anti-Patterns — Explicit Rejection

SVOS ventures **must not** ship with these patterns.

| Anti-pattern | Why rejected | Alternative |
|--------------|--------------|-------------|
| **Dashboard syndrome** | Metrics without user job | Task-first home |
| **Glassmorphism abuse** | Readability loss, trend | Solid surfaces + subtle border |
| **Gradient spam** | AI-generated look | Single accent, semantic color |
| **Excessive animation** | Distraction, perf cost | Motion for state change only |
| **AI-generated appearance** | Generic, untrustworthy | Intentional typography + spacing |
| **Cluttered interfaces** | Cognitive overload | Progressive disclosure |
| **Template look** | No brand memory | Venture-specific accent only |
| **Dark patterns** | Trust destruction | Transparent monetization |

`ux` capability signs off before V3 release.
